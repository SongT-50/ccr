# GeekNews 포스트

## 제목

Show: CCR - LLM이 자기 결과물 리뷰할 때 놓치는 오류를 잡아주는 작은 도구를 만들어봤습니다

## 본문

LLM으로 코딩하다가 "방금 쓴 거 리뷰해줘"라고 하면, 자기가 쓴 걸 잘 못 찾더라고요. 왜 그런지 궁금해서 찾아보니 앵커링 편향이라는 게 있었습니다. 같은 대화에서 리뷰하면 자기 맥락에 갇혀서 오류를 놓친다고요.

그래서 간단한 아이디어를 시도해봤습니다. 같은 세션에서 리뷰하는 대신, 아예 새로운 API 세션을 여러 개 띄워서 결과물만 보고 리뷰하게 하는 거예요. 대화 히스토리가 없으니 편향도 없습니다.

```bash
pip install ccr-review
ccr review mycode.py
```

3명의 독립 리뷰어가 각각 보고, 디렉터가 결과를 모아줍니다. 30초 정도, 비용은 Haiku 기준 약 20원입니다.

실험해보니 (360회 리뷰):
- F1: 28.6% vs 같은 세션 24.6% (p=0.008)
- 치명적 오류 탐지: 40% vs 29%
- 같은 세션에서 반복해도 효과 없음 — 세션 격리가 핵심이었습니다

Claude, GPT, Gemini 다 됩니다. 모델 문제가 아니라 구조적인 문제더라고요.

공부하면서 논문도 3편 써봤는데, 혹시 이 분야 잘 아시는 분이 계시면 피드백 주시면 정말 감사하겠습니다:
