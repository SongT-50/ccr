# GeekNews 포스트

## 제목

Show: CCR - LLM이 자기 결과물을 리뷰하면 64.5%의 오류를 놓깁니다. 세션 격리로 해결하는 오픈소스 도구

## 본문

LLM에게 "방금 쓴 거 리뷰해줘"라고 하면, 자기가 쓴 것에 앵커링되어 오류를 잘 못 찾습니다. Tsui(2025)의 측정에 따르면 64.5%를 놓칩니다.

CCR(Cross-Context Review)은 구조적으로 간단한 접근입니다. 같은 세션에서 리뷰하는 대신, 완전히 독립된 API 세션 N개를 띄워서 결과물만 보고 리뷰하게 합니다. 대화 히스토리가 없으니 앵커링 편향도 없습니다.

```bash
pip install ccr-review
ccr review mycode.py
```

3명의 독립 리뷰어 + 디렉터가 결과를 통합합니다. 약 30초, 비용은 Haiku 기준 ~$0.02입니다.

**실험 결과** (360회 리뷰, 30개 결과물, 150개 정답 오류):

- F1: 28.6% vs 같은 세션 리뷰 24.6% (p=0.008)
- 치명적 오류 탐지율: 40% vs 29%
- 같은 세션에서 리뷰를 반복해도 효과 없음 (SR2 ~ SR, p=0.11)
- 반복이 아니라 세션 격리가 핵심 메커니즘

모델에 무관하게 작동합니다 (Claude, GPT, Gemini). 이 편향은 특정 모델의 문제가 아니라 구조적인 문제이기 때문입니다.

논문 3편 기반으로 만들었습니다:
- **CCR**: 세션 격리 효과 실증 — https://arxiv.org/abs/2603.12123
- **D-CCR**: 다회차 리뷰가 오히려 해롭다는 것을 증명 (False Positive +62%) — https://arxiv.org/abs/2603.16244
- **HCCA**: 계층적 다중 에이전트 검증 아키텍처 — https://arxiv.org/abs/2603.21454
